# LRN: Top 10 Performance Tuning Tips for the SAS Viya Platform

In this Learn Session we share tuning tips for SAS Viya to avoid common bottlenecks and performance issues. We touch on recommended practices for critical areas, such as architecture, Viya services, as well as analytic runtimes like SAS Cloud Analytics Service and SAS Programming Runtime Environment. We wrap up showing how to monitor the environment to ensure that the tuning is working as expected.

The Top 10 tips:

1.  [Better storage configuration](/01-Better-Storage-Config.md)
1.  [Find faster SASWORK](/02-Find-Faster-Saswork.md)
1.  [SPRE Reusable Compute](/03-Spre-Reusable-Compute.md)
1.  [SAS Code Analyzer (SCAPROC)](/04-ScaProc.md)
1.  [WLM Elastic Autoscaling](/05-Wlm-Elastic-Autoscaling.md)
1.  [WLM Can Match Jobs to Hardware](/06-Wlm-Match-Jobs.md)
1.  [SAS Data Velocity](/07-Sas-Data-Velocity.md)
1.  [Monitor SAS Viya in Kubernetes](/08-Monitor-Sas-Viya.md)

1.  [Adjust SAS Viya Analytic Runtimes](/10-Adjust-Viya-Runtimes.md)
